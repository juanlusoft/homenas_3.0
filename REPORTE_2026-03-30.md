## Sistema de actualizaciones OTA (v6.4.1 → v6.5.0)

### Problema
El botón "Buscar actualizaciones" en Sistema no funcionaba y no había forma de actualizar el dashboard remotamente desde los NAS en producción.

### Solución implementada
- `GET /api/system/git-check` — comprueba commits pendientes en `origin/main` via `git fetch` + `git log HEAD..origin/main`
- `POST /api/system/git-update` — ejecuta `git pull` + `pnpm install` + `pnpm build` + reinicio del servicio
- Fix crítico: el endpoint usaba `process.cwd()` que en systemd no apunta al repo → cambiado a ruta absoluta desde `import.meta.url`
- **OTA incluye build del frontend** — antes solo hacía `git pull`, el frontend (`dist/`) nunca se actualizaba → ahora `pnpm run build` forma parte del proceso
- `CI=true` en `pnpm install` para evitar el error de no-TTY en entorno systemd

### Badge de actualización en sidebar
- Al hacer login y cada hora, el frontend consulta `/api/system/git-check`
- Si hay commits pendientes → aparece un badge pulsante teal encima del usuario con "Actualización disponible"
- El badge navega directamente a la página Sistema
- La página Sistema muestra la lista de commits pendientes

## Fix: guardar IP estática en Red (v6.4.2 → v6.4.8)

### Problema
El botón "Guardar" en el modal de configuración de red no hacía nada (dos bugs):
1. URL incorrecta en el frontend (`/network/interfaces/:id` → debía ser `/network/:id/config`)
2. `nmcli con mod <interfaz>` falla cuando el nombre del perfil de conexión de NetworkManager no coincide con el nombre de la interfaz (ej: interfaz `eth0` con perfil "Wired connection 1")

### Solución
- Frontend: URL corregida a `PUT /network/${editIface}/config`
- Backend: consulta `nmcli -g NAME,DEVICE con show` para resolver el nombre real del perfil antes de modificarlo
- Frontend: añadido estado de guardado (`...`) y mensaje de error en el modal si falla

## Fix: hora en gráficos del dashboard (v6.4.9)

### Problema
El eje X de los gráficos mostraba `34:37`, `35:01`, etc. que parecía tiempo de uptime pero era la hora real sin las horas (formato `MM:SS` en lugar de `HH:MM:SS`).

### Solución
`formatTime()` en `MetricsChart.tsx` y `NetworkChart.tsx` actualizado de:
```ts
{ minute: '2-digit', second: '2-digit' }
```
a:
```ts
{ hour: '2-digit', minute: '2-digit', second: '2-digit' }
```

## Fix: servicio systemd no arrancaba (v6.4.9)

### Problema
Tras actualizar con `CI=true pnpm install`, el servicio fallaba con `tsx: not found`. El `ExecStart` usaba `npx tsx` pero `tsx` no estaba en el PATH de systemd.

### Solución
`ExecStart` cambiado a ruta directa:
```
/usr/bin/node /opt/homepinas-v3/node_modules/tsx/dist/cli.mjs server/index.ts
```
También actualizado en `install.sh` para futuras instalaciones.

## Almacenamiento: port desde repo anterior (v6.4.4 → v6.4.6)

### Bloque 1 — Discos disponibles y montaje (v6.4.4)
- `GET /storage/available-disks` — detecta discos sin usar via `lsblk` JSON
- `POST /storage/add-to-pool` — formato GPT + ext4 + mount + MergerFS remount,add + fstab
- `POST /storage/mount-standalone` — monta disco independiente en `/mnt/<nombre>`

### Bloque 3 — Badblocks y Cache Mover (v6.4.6)

## Active Backup: generación de agentes (v6.5.2)

### Descripción
Nuevo sistema para generar instaladores de backup para equipos remotos (Windows, macOS, Linux).

### Backend
`GET /active-backup/agent/generate/:platform?backupType=<tipo>`

Parámetros:
- `platform`: `windows` | `mac` | `linux`
- `backupType`: `full` | `incremental` | `folders`

El endpoint:
1. Genera un token único de 32 bytes
2. Pre-registra el dispositivo como pending en el NAS
3. Devuelve el script con todos los valores embebidos

### Scripts generados

| Plataforma | Formato | Mecanismo backup | Scheduler |
|-----------|---------|-----------------|-----------|
| Windows | `.ps1` (PowerShell) | `robocopy` → SMB share `\\NAS\active-backup\<id>` | Tarea Programada 02:00 |
| macOS | `.sh` (bash) | `rsync` → SSH | launchd plist 02:00 |
| Linux | `.sh` (bash) | `rsync` → SSH | cron 02:00 |

### Tipos de backup
| Tipo | Windows | macOS | Linux |
|------|---------|-------|-------|
| **Disco completo** | `C:\` | `/` | `/` |
| **Incremental** | `%USERPROFILE%` + flag `/XO` (solo cambios) | `/Users/$USER` | `$HOME` |
| **Carpetas** | Documents, Desktop, Pictures | Documents, Desktop, Pictures, Movies | Documents, Desktop, Pictures, Videos |

### Frontend
- El botón "Descargar agente" abre un modal con dos pasos:
  1. Selector de tipo de backup (3 opciones con descripción)
  2. Selector de SO (descarga el instalador correspondiente)
- El dispositivo aparece en "Pendientes" del dashboard para que el admin lo apruebe
